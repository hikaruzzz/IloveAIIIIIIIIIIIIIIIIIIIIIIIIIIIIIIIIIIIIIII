# 复盘查询

当用户调用 `/share-review-query` 或提及「复盘」「查询分析记录」「历史分析」「之前分析的」时，按以下规则执行。

## 数据来源

- 读取 `data/analysis/` 目录下所有 JSON 文件
- 同时也读取 `data/strategies/` 目录下所有策略文件（用于关联策略信息）

## 操作流程

### 1. 查询历史分析记录

支持以下查询维度，可组合使用：

**按时间查询：**
- 指定日期：`2026-06-19` — 查询当天的所有分析
- 时间范围：`2026-06-15 ~ 2026-06-19` — 查询该时间段内的分析
- 最近 N 条：`最近3次` — 查询最近的 N 条记录

**按标的查询：**
- 股票名称：`贵州茅台`
- 股票代码：`600519`
- 部分匹配：`茅台` — 模糊搜索股票名称

**按策略查询：**
- 策略名称或 slug：`ma-cross`、`均线交叉`

**按结果查询：**
- 入场信号：`建议买入`、`入场`、`看多`
- 出场信号：`建议卖出`、`出场`、`看空`

### 2. 展示查询结果

**列表模式（默认）：**
以表格形式展示匹配的记录：

| 时间 | 股票 | 代码 | 策略 | 信号 | 建议 |
|------|------|------|------|------|------|
| 06-19 15:00 | 贵州茅台 | 600519 | 均线交叉 | 入场 | 建议买入 |
| 06-18 10:30 | 宁德时代 | 300750 | 参考期货 | 观望 | 继续持有 |

**详情模式：**
- 用户选择某条记录后，展示完整分析报告
- 格式化输出数据面、技术面、消息面、策略匹配详情

### 3. 复盘分析

当用户要求「复盘」时，执行以下分析：

- 对比分析结果与后续实际走势
- 统计策略信号的准确率
- 找出信号与实际的偏差
- 总结策略的有效性和改进方向

复盘需要用户提供分析后的实际走势数据。输出格式：

```
📈 复盘报告

标的：{股票名}（{股票号码}）
分析时间：{原分析时间}
复盘时间：{当前时间}

━━━ 原分析结论 ━━━
（原策略匹配结果摘要）

━━━ 实际走势 ━━━
（用户提供的实际走势数据）

━━━ 偏差分析 ━━━
- 信号准确率
- 主要偏差来源
- 策略表现评估

━━━ 改进建议 ━━━
- 策略参数调整建议
- 风控优化建议
```

### 4. 统计分析（可选）

当用户要求统计数据时：

- 统计某时间段内的分析次数
- 统计各策略的使用频率
- 统计信号的准确率（需要实际走势数据）
- 按标的统计关注度

## 数据展示约定

- 时间统一使用北京时间（UTC+8）显示
- 金额单位：A 股用人民币、港股用港币、美股用美元
- 列表模式下，每条记录一行，不超过 6 列
- 详情模式下，完整展示所有字段

## 关键约束

1. 查询不到结果时，明确告知用户并建议调整查询条件
2. 复盘分析时，如果缺少实际走势数据，提醒用户补充
3. 复盘结论为分析参考，不构成投资建议
