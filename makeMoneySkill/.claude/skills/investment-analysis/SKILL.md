# 投资分析

当用户调用 `/share-investment-analysis` 或提及「分析行情」「投资分析」「用策略分析」「帮我分析」时，按以下规则执行。

## 数据存储

- 分析结果保存在 `data/analysis/` 目录下
- 每条分析记录一个 JSON 文件，文件名格式：`{YYYYMMDD-HHmmss}-{标的代码}.json`，如 `20260619-150000-XAUUSD.json`

## 操作流程

### 0. 前置检查：确认默认获取渠道

在开始分析之前，**必须**先检查默认获取渠道：

- 读取 `data/sources/_default.json`
- 如果文件不存在或为空，**立即终止**，告知用户：
  > ⛔ 尚未设置默认数据获取渠道，无法进行投资分析。
  > 请先使用 `/share-data-source-manager` 设置默认获取渠道。
- 如果存在，读取对应数据源的完整信息，记录其 `name`、`slug`、`type`
- 本次分析**只能**从该渠道获取数据，不可使用其他渠道
- 在分析报告中明确标注数据来源

### 1. 自动筛选适用策略（全量分析）

**不要询问用户选择策略。** 流程如下：

1. 读取 `data/strategies/` 下所有策略文件
2. 根据标的物的市场类型，判断每条策略是否适用：

   **适用性判断规则：**
   - 策略的 `market` 字段包含标的所属市场 → **适用**
   - 策略的 `market` 字段标注「通用」→ **适用**
   - 策略的 `market` 字段不包含标的所属市场且非通用 → **不适用**，跳过
   - 如果策略有特殊前置条件（如「参考期货分析」需要标的物有对应期货），需额外检查

3. 展示筛选结果（哪些适用、哪些不适用及原因），然后对所有适用策略并行分析
4. 如果没有任何策略适用，告知用户并终止

### 2. 收集标的信息

- 从用户输入中提取标的名称和代码
- 如果用户只提供了名称或代码，尝试推断补充
- 标的可以是股票、ETF、期货、贵金属、加密货币等，不限于股票

### 3. 获取行情数据

#### 3a. 判断数据源类型

读取默认数据源的 `type` 字段：

**如果 `type` 为 `skill`：**
- **直接调用对应的 skill 获取数据**，不再向用户索要数据
- 调用方式：使用 `Skill` 工具，skill 名称为数据源的 `url` 去掉前导 `/`，例如 `url: "/share-a-stock-data"` → 调用 `Skill("a-stock-data")`
- 将标的名称/代码作为 args 传入：`Skill("a-stock-data", "获取 伦敦现金 XAUUSD 的行情数据")`
- 如果调用失败或 skill 不存在，告知用户并询问是否手动提供数据

**如果 `type` 为 `api`、`website`、`sdk`、`data-vendor` 等非 skill 类型：**
- 提醒用户数据来源限定为「{默认渠道名称}」
- 向用户收集以下三类数据（用户可分批提供，不必一次性全部给出）：

**数据面（基本面）：**
- 当前价格（必须）
- PE、PB、ROE（可选，股票适用）
- 市值（可选）
- 其他与标的类型相关的基本面指标

**技术面（用户观察到的技术指标）：**
- 当前各均线位置及交叉情况
- MACD 状态（金叉/死叉/钝化）
- RSI、KDJ 等指标值（如有）
- K 线形态、成交量变化等
- 用户可直接粘贴截图描述或口头描述技术面情况

**消息面：**
- 近期相关新闻
- 公告、财报
- 行业动态
- 市场情绪判断

#### 3b. 数据汇总

将获取到的数据（无论来自 skill 调用还是用户提供）整理为统一的三面结构（数据面、技术面、消息面），供后续策略匹配使用。

### 4. 全量策略匹配分析

> ⚠️ **关键：对每一条适用的策略，都要独立完成匹配分析，不可跳过任何一条。**

对每条适用策略执行：

1. 将该策略的入场/出场规则与已获取的数据逐一比对
2. 标注每条规则是否满足（✅/❌）
3. 统计入场信号满足数和出场信号满足数
4. 生成该策略的综合建议（看多/看空/观望）
5. 结合该策略自身的风控设置给出操作建议

**不适用策略的处理：**
- 对于不适用的策略，在结果中标注「不适用」及具体原因
- 不需要进行详细匹配

### 5. 综合结论

全量策略分析完成后，汇总：

- 共 N 条策略，M 条适用，其中 X 条看多、Y 条看空、Z 条观望
- 如果多条策略方向一致，标注「多策略共振，信号增强」
- 如果策略之间方向矛盾，标注分歧并解释各策略逻辑
- 给出综合判断（仅供参考，不构成投资建议）

### 5. 保存结果

- 汇总所有策略的分析结论，生成 JSON 文件（含 `策略匹配结果` 数组和 `综合结论`）
- 写入 `data/analysis/{YYYYMMDD-HHmmss}-{标的代码}.json`
- 告知用户文件已保存

## 分析框架

分析时遵循以下结构，确保覆盖全面：

```
📊 投资分析报告

标的：{标的名称}（{代码}）
市场类型：{A股/港股/美股/贵金属/加密货币/...}
数据来源：{默认获取渠道名称}（自动获取 / 用户提供）
时间：{分析时间}

━━━ 策略筛选 ━━━
全量策略：N 条
✅ 适用（M 条）：策略A、策略B、...
❌ 不适用（K 条）：策略X（原因）、策略Y（原因）

━━━ 数据面 ━━━
（基本面数据整理）

━━━ 技术面 ━━━
（技术指标解读）

━━━ 消息面 ━━━
（新闻与情绪判断）

━━━ 策略匹配（逐条） ━━━
【策略A】类型：trend | 适用性：✅
  ✅ 已满足的入场条件
  ❌ 未满足的条件
  📌 策略建议

【策略B】类型：custom | 适用性：✅
  ...（同上）

【策略X】类型：xxx | 适用性：❌
  原因：标的物无对应期货品种

━━━ 综合结论 ━━━
多策略共振/分歧分析
最终综合判断

```

## 关键约束

1. **必须通过前置检查**：无默认获取渠道时，终止分析，引导用户去设置
2. **全量策略，不做选择**：不要询问用户「用哪个策略」，直接对所有适用策略做并行分析；不适用策略标注原因
3. **skill 类型数据源自动调用**：当默认数据源 `type` 为 `skill` 时，直接通过 `Skill` 工具调用对应 skill 获取数据，不要向用户索要
4. **每一条适用策略都必须独立分析**：不可合并、不可跳过、不可只选一条
5. 分析完成后必须立即将结果写入 `data/analysis/` 目录
6. 如果用户提供的数据不足以完成分析，明确指出缺失项并询问
7. 分析结论中必须包含风险提示
8. 不可提供具体的买卖建议（合规要求），使用「策略信号显示…」「根据策略规则…」等表述
9. **数据来源强制锁定**：分析全程只能使用默认获取渠道，不可建议或接受用户从其他渠道获取数据
