# 投资分析

当用户调用 `/share-investment-analysis` 或提及「分析行情」「投资分析」「用策略分析」「帮我分析」时，按以下规则执行。

## 操作流程

### 0. 前置检查：确认默认获取渠道

在开始分析之前，**必须**先检查默认获取渠道：

- 读取 `data/sources/_default.json`
- 如果文件不存在或为空，**立即终止**，告知用户：
  > ⛔ 尚未设置默认数据获取渠道，无法进行投资分析。
  > 请先使用 `/share-data-source-manager` 设置默认获取渠道。
- 如果存在，读取对应数据源的完整信息，记录其 `name`、`slug`、`type`
- 本次分析**只能**从该渠道获取数据，不可使用其他渠道
- 在分析报告中明确标注数据来源

### 1. 收集标的信息 + 筛选适用策略

**不要询问用户选择策略。** 流程如下：

1. 从用户输入中提取标的名称和代码
2. 读取 `data/strategies/` 下所有策略文件
3. 根据标的物的市场类型，判断每条策略是否适用：

   **适用性判断规则：**
   - 策略的 `market` 字段包含标的所属市场 → **适用**
   - 策略的 `market` 字段标注「通用」→ **适用**
   - 策略的 `market` 字段不包含标的所属市场且非通用 → **不适用**，跳过
   - 如果策略有特殊前置条件（如「参考期货分析」需要标的物有对应期货），需额外检查

4. 展示筛选结果（哪些适用、哪些不适用及原因）
5. 如果没有任何策略适用，告知用户并终止

### 2. 分析数据需求（按策略倒推）

> ⚠️ **关键：不要盲目获取所有数据。先分析适用策略需要什么，再精准获取。**

对每条适用策略，解析其 `indicators`、`entryRules`、`exitRules` 字段，提取所需的全部数据项：

**提取规则：**
- `indicators` 中每个指标 → 对应数据项（如 `MA(5)` → MA5、`MACD` → DIF/DEA/柱、`RSI(14)` → RSI）
- `entryRules` / `exitRules` 中提及的数据 → 对应数据项（如「成交量放大」→ 成交量、「基差」→ 期货基差、「持仓量」→ OI）
- 基本面数据（价格始终需要，PE/PB/ROE 等视标的类型而定）
- 消息面（始终需要，用于判断市场情绪）

**汇总数据需求清单：**
- 将所有策略的数据需求去重合并，生成统一清单
- 例如：`["当前价格", "MA5", "MA10", "MA20", "MACD(DIF/DEA)", "RSI(14)", "成交量", "COMEX持仓量", "基差"]`
- 展示给用户：「根据 X 条适用策略，需要以下 Y 项数据：...」

### 3. 按需求清单获取数据

根据步骤 2 的数据需求清单，从默认数据源获取数据：

#### 3a. 判断数据源类型

读取默认数据源的 `type` 字段：

**如果 `type` 为 `skill`：**
- **直接调用对应的 skill 获取数据**
- 调用方式：使用 `Skill` 工具，skill 名称为数据源的 `url` 去掉前导 `/`
- 将标的 + 数据需求清单作为 args 传入，明确列出需要的数据项
- 如果调用失败或 skill 不存在，告知用户并询问是否手动提供数据

**如果 `type` 为 `api`、`website`、`sdk`、`data-vendor` 等非 skill 类型：**
- 向用户展示数据需求清单，提醒数据来源限定为「{默认渠道名称}」
- 用户可分批提供数据

#### 3b. 数据汇总

将获取到的数据按三面结构（数据面、技术面、消息面）整理，并额外标注支撑压力位数据。

### 4. 策略匹配 + 支撑压力位预测

#### 4a. 全量策略匹配

> ⚠️ **关键：对每一条适用的策略，都要独立完成匹配分析，不可跳过任何一条。**

对每条适用策略执行：
1. 将该策略的入场/出场规则与已获取的数据逐一比对
2. 标注每条规则是否满足（✅/❌）
3. 统计入场信号满足数和出场信号满足数
4. 生成该策略的综合建议（看多/看空/观望）
5. 结合该策略自身的风控设置给出操作建议

不适用策略标注原因即可，无需详细匹配。

#### 4b. 支撑位与压力位预测

基于获取到的技术面数据，计算并预测关键支撑位和压力位：

**支撑位计算依据（按优先级排列）：**
1. 关键均线位置（MA20 / MA50 / MA200）
2. 近期低点 / 前低
3. 筹码密集区 / 成交量密集区
4. 斐波那契回撤位（38.2%、50%、61.8%）
5. 布林带下轨
6. 整数关口

**压力位计算依据（按优先级排列）：**
1. 关键均线位置（MA20 / MA50 / MA200）
2. 近期高点 / 前高
3. 筹码密集区 / 成交量密集区
4. 斐波那契回撤位（38.2%、50%、61.8%）
5. 布林带上轨
6. 整数关口

**展示格式：**
每个支撑/压力位标注：具体价位、强度（⭐1-5）、计算依据。

**强度判断标准：**
- ⭐⭐⭐⭐⭐：多重依据共振（如 MA200 + 前低 + 整数关口在同一区域）
- ⭐⭐⭐⭐：两个依据重叠
- ⭐⭐⭐：单一较强依据
- ⭐⭐：次要参考位
- ⭐：弱参考（心理关口等）

### 5. 综合结论

全量策略分析完成后，汇总：

- 共 N 条策略，M 条适用，其中 X 条看多、Y 条看空、Z 条观望
- 如果多条策略方向一致，标注「多策略共振，信号增强」
- 如果策略之间方向矛盾，标注分歧并解释各策略逻辑
- 结合支撑压力位，给出关键价位参考
- 给出综合判断（仅供参考，不构成投资建议）

### 6. 保存结果

- 汇总所有分析结论，生成 JSON 文件（含 `数据需求清单`、`策略匹配结果` 数组、`支撑压力位`、`综合结论`）
- 写入 `data/analysis/{YYYYMMDD-HHmmss}-{标的代码}.json`
- 告知用户文件已保存

## 关键约束

1. **必须通过前置检查**：无默认获取渠道时，终止分析，引导用户去设置
2. **全量策略，不做选择**：不要询问用户「用哪个策略」，直接对所有适用策略做并行分析；不适用策略标注原因
3. **先分析数据需求，再获取数据**：先解析适用策略的 indicators/entryRules/exitRules，汇总去重得到数据需求清单，然后按清单精准获取，不要盲目请求所有数据
4. **skill 类型数据源自动调用**：当默认数据源 `type` 为 `skill` 时，直接通过 `Skill` 工具调用对应 skill 获取数据，args 中明确列出数据需求清单
5. **每一条适用策略都必须独立分析**：不可合并、不可跳过、不可只选一条
6. **必须输出支撑压力位**：基于技术面数据计算关键支撑位和压力位，每个价位标注强度和计算依据，至少各 3 个
7. 分析完成后必须立即将结果写入 `data/analysis/` 目录
8. 如果获取的数据不足以完成分析，明确指出缺失项并询问
9. 分析结论中必须包含风险提示
10. 不可提供具体的买卖建议（合规要求），使用「策略信号显示…」「根据策略规则…」等表述
11. **数据来源强制锁定**：分析全程只能使用默认获取渠道，不可建议或接受用户从其他渠道获取数据
