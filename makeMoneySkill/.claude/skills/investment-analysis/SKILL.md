# 投资分析

当用户调用 `/share-investment-analysis` 或提及「分析行情」「投资分析」「用策略分析」「帮我分析」时，按以下规则执行。

## 数据存储

- 分析结果保存在 `data/analysis/` 目录下
- 每条分析记录一个 JSON 文件，文件名格式：`{YYYYMMDD-HHmmss}-{股票号码}.json`，如 `20260619-150000-000001.json`

## 操作流程

### 1. 选择策略

- 首先读取 `data/strategies/` 下所有策略文件
- 列出可用策略供用户选择（名称、类型、市场、描述）
- 用户选择一条策略进行分析

### 2. 收集标的信息

- 询问用户要分析的股票名称和代码
- 如果用户只提供了名称或代码，尝试补充另一项

### 3. 收集行情数据

向用户收集以下三类数据（用户可分批提供，不必一次性全部给出）：

**数据面（基本面）：**
- 当前价格（必须）
- PE、PB、ROE（可选）
- 市值（可选）
- 其他用户认为重要的基本面指标

**技术面（用户观察到的技术指标）：**
- 当前各均线位置及交叉情况
- MACD 状态（金叉/死叉/钝化）
- RSI、KDJ 等指标值（如有）
- K 线形态、成交量变化等
- 用户可直接粘贴截图描述或口头描述技术面情况

**消息面：**
- 近期相关新闻
- 公告、财报
- 行业动态
- 市场情绪判断

### 4. 策略匹配分析

- 将用户提供的数据与所选策略的入场/出场规则逐一比对
- 标注哪些规则已满足、哪些未满足
- 根据匹配结果给出综合建议（买入/卖出/持有观望）
- 结合策略的风控设置给出具体操作建议

### 5. 保存结果

- 汇总所有分析内容，生成 JSON 文件
- 写入 `data/analysis/{YYYYMMDD-HHmmss}-{股票号码}.json`
- 告知用户文件已保存

## 分析框架

分析时遵循以下结构，确保覆盖全面：

```
📊 投资分析报告

标的：{股票名}（{股票号码}）
策略：{策略名称}
时间：{分析时间}

━━━ 数据面 ━━━
（基本面数据整理）

━━━ 技术面 ━━━
（技术指标解读）

━━━ 消息面 ━━━
（新闻与情绪判断）

━━━ 策略匹配 ━━━
✅ 已满足的入场/出场条件
❌ 未满足的条件
📌 综合建议与操作计划

⚠️ 风险提示
```

## 关键约束

1. 分析完成后必须立即将结果写入 `data/analysis/` 目录
2. 如果用户提供的数据不足以完成分析，明确指出缺失项并询问
3. 分析结论中必须包含风险提示
4. 不可提供具体的买卖建议（合规要求），使用「策略信号显示…」「根据策略规则…」等表述
